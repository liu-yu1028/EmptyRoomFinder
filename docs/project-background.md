# 製作動機或目的

RoomRush（空教室查詢系統）是以輔仁大學學生的日常需求為出發點所製作的 Android App。這份文件整理專案的製作動機、目標範圍與預期解決的問題。

---

## 製作動機

大學生的課表經常會出現一到兩小時的空堂。若學生住家距離學校較遠，空堂時間通常不足以往返住處，因此會需要在校園內尋找可以短暫休息、自習、討論或用餐的空間。

然而，學生不一定清楚哪些教室在特定星期與節次沒有被使用，也可能需要花時間一間一間確認。基於這個情境，我們希望建立一個快速、直覺且方便查詢的空教室工具，讓使用者能依照自己的需求找到合適的教室。

---

## 專案目的

本專案以 114 學年度下學期課表為資料基礎，建立一個可查詢空教室的 Android App。使用者可以依照「大樓、星期、時段、樓層」篩選教室，快速查看符合條件的空教室。

除了查詢功能外，系統也提供管理者模式，讓管理者能維護教室資料，例如修改課表狀態、新增教室、刪除教室，以及設定教室類型。

---

## 查詢範圍

目前系統資料來源主要包含：

| 項目 | 範圍 |
|---|---|
| 學校 | 輔仁大學 |
| 大樓 | 聖言樓、進修部大樓 |
| 樓層 | 1 到 6 樓 |
| 資料來源 | 專題整理之 CSV 課表資料 |
| 使用情境 | 查詢空教室、自習、休息、討論、查看教室資訊 |

---

## 專案定位

RoomRush 原本是大一下學期的小組期末專題，後續重新整理為可以放在 GitHub 展示的 Android 學習作品。

這個專案的重點不只是「做出一個 App」，也包含：

- 練習把校園需求轉成 App 功能。
- 理解 Android Activity、ViewModel、Repository、Room Database 的分工。
- 整理舊專題，使其成為更完整、可閱讀、可展示的作品。
- 作為後續鐵人賽文章與學習紀錄的實作案例。
