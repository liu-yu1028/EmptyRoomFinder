# 功能描述

本文件整理 RoomRush 的主要功能需求，分為「使用者模式」與「管理者模式」。

---

## 功能總覽

RoomRush 的核心目標是讓使用者能快速查詢特定條件下的空教室，並讓管理者維護教室與課表資料。

```text
使用者模式：查詢空教室、查看教室詳細資訊
管理者模式：登入、管理課表、設定教室類型、瀏覽與新增教室
```

---

## 使用者模式

### 1. 首頁：選擇查詢大樓

使用者進入 App 後，會先在首頁選擇想查詢的大樓。

| 項目 | 說明 |
|---|---|
| 可選大樓 | 聖言樓、進修部大樓 |
| 主要操作 | 選擇大樓後進入查詢結果頁 |
| 對應頁面 | `MainActivity` |

---

### 2. 查詢結果頁：篩選空教室

使用者可以依照以下條件篩選空教室：

| 條件 | 說明 |
|---|---|
| 星期 | 星期一到星期五 |
| 時段 | 第一節到第八節，包含中午時段 |
| 樓層 | 可選單一樓層，也可選全部樓層 |
| 查詢結果 | 依條件即時顯示可使用教室 |

查詢結果會以列表呈現，並依樓層分組，方便使用者快速瀏覽。

---

### 3. 教室詳細頁：查看教室資訊

使用者在查詢結果中點選教室後，可以進入教室詳細頁，查看：

- 教室名稱
- 教室類型
- 是否可以飲食

飲食規則目前依教室類型判斷：

| 教室類型 | 是否可飲食 |
|---|---|
| 一般教室 | 可飲食 |
| 電腦教室 | 不可飲食 |
| 製圖教室 | 不可飲食 |
| 實驗室 | 不可飲食 |
| 研討室 | 不可飲食 |

---

## 管理者模式

### 1. 管理者登入

管理者需先登入才能進入管理功能，一般使用者不能直接進入管理介面。

| 功能 | 說明 |
|---|---|
| 登入 | 輸入指定帳號密碼後進入管理系統 |
| 登出 | 離開管理頁面時清除登入狀態 |
| 對應頁面 | `LoginActivity`、`ManagerActivity` |

> 目前登入功能屬於課程專題用的簡化設計，若要正式使用，需重新設計安全的驗證機制。

---

### 2. 管理系統首頁

管理者登入後可以進入管理系統首頁，並選擇以下功能：

| 功能 | 說明 |
|---|---|
| 課表管理 | 修改教室有課時段、刪除教室 |
| 教室詳細設定 | 修改教室類型與飲食規則顯示 |
| 瀏覽教室狀態 | 查看所有教室資料與課表狀態 |

---

### 3. 課表管理

管理者可以透過下拉選單選擇：

- 大樓
- 樓層
- 教室名稱
- 星期
- 時段
- 是否有課

並執行：

- 修改教室課表狀態
- 刪除教室資料

---

### 4. 教室詳細設定

管理者可以選擇特定教室，修改該教室的類型。飲食規則會依照教室類型自動判斷，而不是讓管理者手動任意調整。

---

### 5. 瀏覽教室狀態與新增教室

管理者可以查看所有教室資料，包含：

- 教室名稱
- 教室類型
- 是否可飲食
- 有課時段

也可以新增教室，並設定教室名稱與教室類型。新增後，相關下拉選單需能依資料更新。

---
